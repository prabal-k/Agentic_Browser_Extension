### Graph Architecture: 13 Nodes

| Node | Cognitive Function | LLM Call? | Interrupt? |
|---|---|---|---|
| `analyze_goal` | Parse user goal → structured Goal | Yes (reasoning) | No |
| `create_plan` | Generate step-by-step plan | Yes (reasoning) | No |
| `critique_plan` | Self-critique the plan | Yes (reasoning) | No |
| `reason` | ReAct + CoT reasoning about current state | Yes (reasoning) | No |
| `decide_action` | Select browser action via tool calling | Yes (action) | No |
| `confirm_action` | Ask user to confirm action | No | **Yes (interrupt)** |
| `execute_action_node` | Send action to browser, receive result | No | **Yes (interrupt)** |
| `observe` | Update memory with action outcome | No | No |
| `evaluate` | Assess action result vs goal | Yes (reasoning) | No |
| `self_critique_action` | Route: continue / retry / re-plan / done | No | No |
| `handle_retry` | Choose different strategy after failure | Yes (reasoning) | No |
| `ask_user_node` | Ask clarification question | No | **Yes (interrupt)** |
| `finalize` | Generate task summary | No | No |

### 5 Conditional Routing Functions

| Router | After Node | Routes To |
|---|---|---|
| `route_after_critique` | critique_plan | create_plan (re-plan) or reason (proceed) |
| `route_after_reasoning` | reason | decide_action, ask_user_node, or finalize |
| `route_after_decision` | decide_action | confirm_action, execute, ask_user, or finalize |
| `route_after_confirm` | confirm_action | execute (confirmed) or reason (rejected) |
| `route_after_self_critique` | self_critique_action | reason, create_plan, handle_retry, or finalize |

### 3 Interrupt Points (LangGraph interrupt())

1. **confirm_action**: Pauses to show user the planned action, waits for Yes/No
2. **execute_action_node**: Pauses to send action to browser, waits for ActionResult + new DOM
3. **ask_user_node**: Pauses to ask clarification question, waits for user input

### Safety Features

- Max iteration guard (default 25) prevents infinite loops
- Max re-plan limit (3) prevents plan thrashing
- Retry context tracks failed strategies to force new approaches
- Confidence threshold controls when to ask user vs auto-execute

## Design Decisions

1. **Separate reasoning vs action LLMs**: Reasoning nodes use temperature=0.4 without tool binding (creative thinking). Action nodes use temperature=0.1 with tool binding (deterministic selection).

2. **JSON structured output, not free text**: All LLM responses are parsed as JSON. This makes output reliable and machine-parseable. Fallbacks handle parse errors gracefully.

3. **3 interrupt points, not more**: Minimized interrupt points to keep the graph simple. Only pause for: user confirmation, browser execution, and clarification questions.

4. **Self-critique without LLM call**: The `self_critique_action` node uses simple heuristics (step completion, evaluation results) rather than calling the LLM. This keeps the execution loop fast. Deep critique is reserved for plan-level critique.

5. **Routing priority order**: In `route_after_decision`, ASKING_USER is checked before DONE to prevent the agent from prematurely finalizing when it needs user input.
